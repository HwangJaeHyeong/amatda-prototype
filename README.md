# 아맞다! 🎒 — 프로토타입

> 스마트한 준비물 리스트 추천·알림 어시스턴트 (5조: 송단미·이수진·이현빈)

프로젝트 브리프(`TalkFile_프로젝트브리프_이현빈_v.1.docx`) 기반 **클릭 가능한 동작 프로토타입**입니다.
iOS 네이티브가 최종 목표지만, 빠른 검증을 위해 **빌드 없이 브라우저에서 바로 돌아가는** iPhone UI 웹 프로토타입으로 제작했습니다.

## 실행 (실제 AI·날씨 연동 버전)
실시간 날씨(위치 권한)와 실제 AI 추천(FactChat 프록시)을 쓰려면 **로컬 서버로** 띄워야 합니다.
```
cd ddudugo-prototype
node server.js          # → http://localhost:8000
```
브라우저에서 **http://localhost:8000** 접속. 위치 권한을 허용하면 실시간 날씨가 반영됩니다.

> 정적 미리보기만 필요하면 `open index.html`(file://)도 되지만, 이 경우 위치·AI 연동은 비활성화되고 **규칙 엔진/모의 날씨로 폴백**합니다.

### 🔑 AI 추천 키 설정 (FactChat / Mindlogic)
- 무료 키 발급 후 서버에 전달하는 방법 (둘 중 하나):
  - 파일: `ddudugo-prototype/.factchat_key` 에 키 한 줄 저장
  - 환경변수: `FACTCHAT_KEY=발급키 node server.js`
- 엔드포인트: `https://factchat-cloud.mindlogic.ai/v1/api/openai/chat/completions` (OpenAI 호환), 모델 `gpt-5-chat-latest`
- 키가 없으면 자동으로 **규칙 엔진**으로 폴백 (앱은 그대로 동작)

> ⚠️ **보안**: 현재 `.factchat_key`에 데모용으로 노출됐던 키가 들어 있습니다. **꼭 폐기하고 새 키로 교체**하세요. 키는 서버에서만 사용되고 브라우저로 전송되지 않습니다.

> ❌ **GPT Plus 구독으로는 불가** — ChatGPT Plus는 API 접근을 포함하지 않습니다(API는 별도 유료). 무료로 쓰려면 FactChat·Gemini 같은 무료 키를 사용하세요.

## 핵심 흐름
### 1. 최초 실행 — 취향분석 온보딩 (쇼핑몰 앱 스타일)
- 라이프스타일 키워드 카드 멀티 선택 (직장인/대학생/운동/반려동물/렌즈·안경/약 챙김 등)
- **자주 깜빡하는 물건** 선택 → 이후 ⚠️로 강조하고 알림 우선순위 부여
- **덜렁거림 정도** 선택 → 잔소리 강도·하루 알림 횟수 자동 조정
- AI 페르소나 선택 (걱정많은 엄마/담임쌤/츤데레 친구/집사로봇/훈련소 조교)
→ 선택 결과로 루틴·오늘 일정·준비물을 자동 세팅

### 2. 일정 추가 — 은행 챗봇 스타일 대화형
- **일정(필수)** 입력 → 봇이 **장소(선택)** → **목적(선택)** 을 순서대로 물어봄
- 분석 후 봇이 준비물을 **퀵리플라이 칩**으로 제시 → **탭 한 번으로 담기/빼기**
- "전체 담기" / "완료" 한 번으로 일정 등록

### 3. 반복 일정 학습
- 같은 카테고리 일정을 2번 이상 추가하면 → 홈에서 **"루틴으로 등록할까요?"** 제안
- 등록 시 `루틴` 탭에 🧠 학습됨 배지로 저장, 다음부터 자동 준비물 템플릿화

## ☁️ Netlify 배포

구성: 정적 화면(`index.html`) + 서버리스 함수(`netlify/functions/recommend.js`, `health.js`). 키는 **환경변수 `FACTCHAT_KEY`** 로만 사용되고 브라우저로 전송되지 않습니다. 설정은 `netlify.toml` 참고.

> ⚠️ 먼저 **FactChat에서 키를 새로 발급**하세요(기존 키는 노출됨). 새 키를 아래 `FACTCHAT_KEY`에 넣습니다.

### CLI로 배포 (이 프로젝트는 git repo가 아니므로 CLI 권장)
```bash
npm i -g netlify-cli           # 1) CLI 설치
cd ddudugo-prototype
netlify login                  # 2) 브라우저 로그인 (세션에선 ! netlify login)
netlify deploy --prod          # 3) 새 사이트 생성 + 배포 (netlify.toml 자동 인식)
netlify env:set FACTCHAT_KEY '새로_발급한_키'   # 4) 키를 환경변수로 등록
netlify deploy --prod          # 5) 키 반영해 재배포
```
배포되면 `https://<사이트명>.netlify.app` 에서 동작합니다. (`/api/recommend`, `/api/health` 는 함수로 자동 연결)

### 또는 웹 UI
1. app.netlify.com → Add new site → 폴더(ddudugo-prototype) 업로드
2. Site configuration → Environment variables → `FACTCHAT_KEY` 추가
3. Functions 디렉터리/리다이렉트는 `netlify.toml` 이 자동 처리 → 재배포

### 키가 없으면?
함수가 `NO_KEY`를 반환하고 앱은 **규칙 엔진으로 폴백**합니다(앱은 정상 동작, AI 추천만 비활성).
